Главное за 2 минуты

В каждом билете 3 вопроса. Вопросы 1 и 2 — устно (рассказать + нарисовать на листке). Вопрос 3 — собрать схему в Tinkercad на компе и вставить код. Инет есть → открываешь эту шпору и делаешь по шагам.

Что отвечать устно (В.1 и В.2)

Тебя попросят: нарисовать обозначение таблицу истинности рассказать как работает. Всё это есть во вкладках «В.1» и «В.2». Рисуешь на листке прямоугольник со значком и таблицу — этого хватает.

Как сдать практику (В.3) — план

  1. Заходишь на tinkercad.com → Circuits → Create new Circuit
  2. Находишь свой билет во вкладке «Билеты 1–30» → там написано, какая это задача
  3. Открываешь нужную задачу во вкладке «В.3» → там схема, какие детали, куда провода, и готовый код
  4. Собираешь схему, жмёшь Code → Text, стираешь всё, вставляешь код
  5. Жмёшь Start Simulation → проверяешь что работает
Если совсем горит: вопрос 3 одинаковый для многих билетов. Выучи сборку 3 базовых схем (светодиод+кнопка, фоторезистор, потенциометр) — они закрывают больше половины билетов.

Базовые слова, чтобы не путаться

  • Светодиод (LED) — лампочка. Длинная ножка (+) к пину, короткая через резистор на GND.
  • Резистор — обязателен с диодом (220 Ом), чтобы не сгорел.
  • Кнопка (тактовая) — замыкает цепь пока нажата.
  • Фоторезистор — меняет сопротивление от света (темно → одно, светло → другое).
  • Потенциометр — крутилка, плавно меняет напряжение.
  • GND — минус (земля). 5V — плюс. A0–A5 — аналоговые входы. 0–13 — цифровые пины.

Вопрос 1 — Логические элементы

Принцип: «n И» / «n ИЛИ» отличаются только числом входов. Логика та же. Рисуешь обозначение + таблицу истинности для 2 входов — этого достаточно, объясняешь что при большем числе входов работает так же.

Как отвечать: «Это логический элемент [название]. Обозначается так. На входы подаются 0 или 1, на выходе получаем по правилу… Вот таблица истинности.»

НЕ (NOT, инвертор)

Переворачивает сигнал. Был 0 → стал 1, был 1 → стал 0.

И (AND) · «2 И», «3 И» … «8 И»

Выход = 1, только если ВСЕ входы = 1. Хоть один 0 → выход 0. Значок «&» в прямоугольнике.

Правило для n входов: 1 на выходе только когда все n входов = 1.

Выход = 1, если хотя бы один вход = 1. Все 0 → выход 0. Значок «1» (или «≥1») в прямоугольнике.

Системы счисления (билеты 1, 2)

Двоично-десятичная (BCD)

Каждую десятичную цифру кодируют 4 битами отдельно. Например число 25: цифра 2 = 0010, цифра 5 = 0101 → пишем 0010 0101. Не путать с обычным двоичным переводом всего числа!

Двоично-шестнадцатеричная

16-ричная цифра = ровно 4 бита. 0–9 как есть, дальше A=10, B=11, C=12, D=13, E=14, F=15. Пример: 0xA = 1010, 0xF = 1111. Чтобы перевести двоичное в hex — бьёшь на группы по 4 бита справа.

Вопрос 2 — Цифровые устройства

Тут рассказываешь что это, рисуешь прямоугольник с входами/выходами и (где нужно) таблицу. Коротко и по делу.

Дешифратор (на 2 / 3 / 4 входа, для 7-сегм.)

Превращает двоичный код на входе в активный сигнал на одном из выходов. n входов → 2ⁿ выходов. Например 2 входа → 4 выхода (один из 4 загорается в зависимости от кода 00/01/10/11).

A1A0Активный выход
00Y0
01Y1
10Y2
11Y3

Дешифратор для 7-сегментного индикатора превращает 4-битный код (0–9) в сигналы для 7 сегментов, чтобы показать цифру.

Шифратор (Encoder)

Обратное дешифратору. На входе активен один из 2ⁿ сигналов → на выходе его номер в двоичном коде. 4 входа → 2 выхода.

Мультиплексор (MUX, на 2/4/5/7 входов)

Переключатель: из нескольких входов данных пропускает на единственный выход тот, что выбран адресными линиями. Как «выбор канала».

Демультиплексор (DEMUX, на 4 выхода)

Обратное MUX. Один вход данных направляет на один из нескольких выходов, выбранный адресом.

Полусумматор и сумматор

Полусумматор: складывает 2 бита, даёт сумму S и перенос C. Сумматор (полный): складывает ещё и перенос из предыдущего разряда (3 входа).

ABS (сумма)C (перенос)
0000
0110
1010
1101

Триггеры (запоминают 1 бит)

RS-триггер

S (Set) = установить в 1, R (Reset) = сбросить в 0. S=R=1 — запрещённая комбинация. Хранит состояние, пока не подадут новый сигнал.

SRQ
00хранит
010
101
11запрет

D-триггер

«Защёлка». По фронту такта запоминает то, что на входе D. Q = D в момент тактового импульса.

T-триггер

«Счётный». Если T=1, на каждом такте Q переключается в противоположное (0↔1). Делит частоту на 2.

JK-триггер

Универсальный. J и K как S и R, но комбинация J=K=1 не запрещена — переключает Q в противоположное (как T).

JKQ
00хранит
010
101
11переключ.

Регистры

Параллельный регистр (4/8-бит) — хранит несколько бит, все загружаются/читаются одновременно. Параллельно-последовательный — загрузка всех бит сразу, выдача по одному. Последовательно-параллельный — наоборот: вводишь по биту, читаешь все сразу. Это набор триггеров.

Прочее (микропроцессор, интерфейсы)

  • Микроконтроллер — мини-компьютер на одном чипе: процессор + память + порты ввода-вывода. Управляет устройствами (Arduino — на нём).
  • АЛУ — арифметико-логическое устройство, ядро процессора. Выполняет арифметику (+, −) и логику (И, ИЛИ, НЕ, XOR и сдвиги).
  • Типовая микропроцессорная система — процессор + память (ОЗУ/ПЗУ) + устройства ввода-вывода, связанные шинами (адреса, данных, управления).
  • SPI — быстрый последовательный интерфейс, 4 провода (MOSI, MISO, SCK, SS). Связь чипов «ведущий-ведомый».
  • I2C / TWI — двухпроводной интерфейс (SDA — данные, SCL — такт). Много устройств на 2 проводах по адресам.
  • UART (УАПП) — асинхронный последовательный обмен по 2 линиям (TX/RX), без отдельного такта.
  • TTL — транзисторно-транзисторная логика, технология построения логических элементов на биполярных транзисторах, уровни 0В/5В.

Вопрос 3 — Сборка + Код

Перед всем: tinkercad.com → Circuits → Create new Circuit. Слева панель деталей. Перетаскиваешь Arduino Uno и Breadboard (макетка) на поле. Провод тянешь кликом от ножки к ножке. Код: кнопка Code справа сверху → переключи на Text → сотри пример → вставь мой код → Start Simulation.
🔆 Задача A — Светодиод горит постоянно / мигает (база)

Детали: Arduino, 1 светодиод, 1 резистор 220 Ом.

Сборка:

  • Длинная ножка (+) светодиода → провод на пин 13
  • Короткая ножка (−) → резистор 220 Ом → пин GND

Код (мигание):

void setup() {
  pinMode(13, OUTPUT);
}
void loop() {
  digitalWrite(13, HIGH);
  delay(1000);
  digitalWrite(13, LOW);
  delay(1000);
}

Чтобы просто «горел постоянно» — убери loop-мигание, оставь digitalWrite(13, HIGH).

🔘 Задача B — Кнопка включает светодиод (пока нажата)

Билеты 3, 4. Детали: Arduino, светодиод, резистор 220 Ом, кнопка.

Сборка:

  • Светодиод: (+) → пин 13, (−) → резистор → GND
  • Кнопка: одна ножка → пин 2, та же сторона → резистор 10кОм → GND; другая ножка кнопки → 5V

Код (билет 3 — нажал = горит, билет 4 — нажал = гаснет — см. ниже):

void setup() {
  pinMode(2, INPUT);
  pinMode(13, OUTPUT);
}
void loop() {
  if (digitalRead(2) == HIGH) {
    digitalWrite(13, HIGH);   // нажал — горит
  } else {
    digitalWrite(13, LOW);
  }
}

Для билета 4 (нажал — гаснет): поменяй местами HIGH и LOW внутри if/else.

🔁 Задача C — Кнопка переключает (нажал-вкл, ещё раз-выкл)

Билеты 6, 22. То же подключение, что в задаче B.

int state = LOW;
int last = LOW;
void setup() {
  pinMode(2, INPUT);
  pinMode(13, OUTPUT);
}
void loop() {
  int btn = digitalRead(2);
  if (btn == HIGH && last == LOW) {
    state = !state;
    digitalWrite(13, state);
    delay(200);
  }
  last = btn;
}
🌑 Задача D — Фоторезистор (темно → диод вкл/выкл)

Билеты 1, 2, 23, 26, 27. Детали: Arduino, фоторезистор, резистор 10кОм, светодиод, резистор 220 Ом.

Сборка:

  • Фоторезистор: одна ножка → 5V; другая ножка → одновременно на пин A0 и через резистор 10кОм на GND
  • Светодиод: (+) → пин 13, (−) → резистор 220 → GND

Код (темно → ВКЛючается):

void setup() {
  pinMode(13, OUTPUT);
}
void loop() {
  int light = analogRead(A0);
  if (light < 500) {       // темно
    digitalWrite(13, HIGH);
  } else {
    digitalWrite(13, LOW);
  }
}

Темно → ВЫКЛючается (билеты 2, 27): поменяй HIGH и LOW местами.

🎛️ Задача E — Потенциометр меняет яркость диода

Билеты 5, 28. Детали: Arduino, потенциометр, светодиод, резистор 220 Ом.

Сборка:

  • Потенциометр: крайние ножки → 5V и GND; средняя ножка → пин A0
  • Светодиод: (+) → пин 9 (ШИМ ~), (−) → резистор → GND
void setup() {
  pinMode(9, OUTPUT);
}
void loop() {
  int val = analogRead(A0);        // 0..1023
  int bright = map(val, 0, 1023, 0, 255);
  analogWrite(9, bright);
}
🚦 Задача F — «Бегущий огонь» (3 светодиода по очереди)

⚡ Задача L — Триггеры на кнопках (T / RS / D / JK)

Билеты 7, 8, 9, 10, 11. Детали: Arduino, 1–3 кнопки, 2 светодиода.

Сборка: кнопки на пины 2,3,4 (через 10к на GND, +5V); диоды на 12 и 13 (через 220 на GND).

T-триггер (билеты 7, 10) — 1 кнопка переключает 2 диода:

int state = 0, last = LOW;
void setup() {
  pinMode(2, INPUT);
  pinMode(12, OUTPUT); pinMode(13, OUTPUT);
}
void loop() {
  int b = digitalRead(2);
  if (b == HIGH && last == LOW) { state = !state; delay(200); }
  last = b;
  digitalWrite(12, state);
  digitalWrite(13, !state);
}

RS / D / JK — то же по сути: кнопка(и) задают, какой диод горит. Для билета 8 (нажал — первый горит/второй нет, ещё раз — наоборот) подходит код выше. Для RS/D/JK на 2 кнопки: одна кнопка → диод1 (Set), вторая → диод2 (Reset):

void setup() {
  pinMode(2, INPUT); pinMode(3, INPUT);
  pinMode(12, OUTPUT); pinMode(13, OUTPUT);
}
void loop() {
  if (digitalRead(2) == HIGH) { digitalWrite(12, HIGH); digitalWrite(13, LOW); }
  if (digitalRead(3) == HIGH) { digitalWrite(12, LOW);  digitalWrite(13, HIGH); }
}

Python — где запускать

Скорее всего стоит IDLE или PyCharm/VS Code. Самое простое — IDLE: File → New File → вставь код → F5 (Run). Или онлайн: onlinegdb.com, replit.com.

Android — где запускать

Если Android совсем не идёт: сделай упор на Python-задачи и теорию — там баллы вернее. Android-код ниже даю готовый, но сборка проекта на незнакомом компе может занять время.

11 — класс Persona, дни до дня рождения
from datetime import date

class Persona:
    def __init__(self, fio, birth_month, birth_day, address):
        self.fio = fio
        self.birth_month = birth_month
        self.birth_day = birth_day
        self.address = address

    def days_to_birthday(self):
        today = date.today()
        bd = date(today.year, self.birth_month, self.birth_day)
        if bd < today:
            bd = date(today.year + 1, self.birth_month, self.birth_day)
        return (bd - today).days

p = Persona("Иванов Иван", 12, 31, "Алматы")
print(p.fio, "- дней до ДР:", p.days_to_birthday())

Android — решения

Как пользоваться: в проекте Android Studio два главных файла. res/layout/activity_main.xml — что на экране. MainActivity.kt (или .java) — логика. Ниже для каждой задачи даю оба. Создавай проект как Empty Views Activity.
1 — Кнопка показывает сообщение (Toast)

activity_main.xml:

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical"
    android:gravity="center">
    <Button
        android:id="@+id/btn"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Нажми меня" />
</LinearLayout>

MainActivity.kt:

import android.os.Bundle
import android.widget.Button
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        val btn = findViewById<Button>(R.id.btn)
        btn.setOnClickListener {
            Toast.makeText(this, "Кнопка нажата!", Toast.LENGTH_SHORT).show()
        }
    }
}

import android.net.Uri
import android.os.Bundle
import android.widget.MediaController
import android.widget.VideoView
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        val video = findViewById<VideoView>(R.id.video)
        // положи video.mp4 в папку res/raw/
        val uri = Uri.parse("android.resource://$packageName/${R.raw.video}")
        video.setVideoURI(uri)
        val mc = MediaController(this)
        video.setMediaController(mc)
        mc.setAnchorView(video)
        video.start()
    }
}

Создай папку res/raw и положи туда video.mp4.

import android.os.Bundle
import android.widget.ImageView
import android.widget.SeekBar
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        val img = findViewById<ImageView>(R.id.img)
        val seek = findViewById<SeekBar>(R.id.seek)
        seek.max = 255
        seek.progress = 255
        seek.setOnSeekBarChangeListener(object : SeekBar.OnSeekBarChangeListener {
            override fun onProgressChanged(s: SeekBar?, p: Int, b: Boolean) {
                img.imageAlpha = p   // прозрачность 0..255
            }
            override fun onStartTrackingTouch(s: SeekBar?) {}
            override fun onStopTrackingTouch(s: SeekBar?) {}
        })
    }
}

Теория · 60 вопросов

Кратко и простым языком. Жми вопрос — раскроется ответ.

🐍 Python (1–40)

1. Типы данных, операторы, переменные, константы
Типы: int (целые), float (дробные), str (строки), bool (True/False), list, tuple, dict, set. Переменная — имя для значения (x = 5). Константа — переменная, которую договорились не менять (пишут БОЛЬШИМИ: PI = 3.14). Операторы: арифметические (+ - * / // % **), сравнения (== != < >), логические (and or not).
2. Операторы ветвления, выбора, цикла
Ветвление: if / elif / else. Выбор (с Python 3.10): match / case. Циклы: for (по элементам/range) и while (пока условие истинно). Управление: break (выход), continue (следующая итерация).
3. Массивы одномерные и многомерные
В Python роль массива играет список (list). Одномерный: a = [1,2,3]. Многомерный — список списков: m = [[1,2],[3,4]], доступ m[0][1] = 2. Настоящие массивы — через модуль array или numpy.
4. Строки. Методы для работы со строками
Строка — последовательность символов. Методы: upper(), lower(), strip() (убрать пробелы), split() (разбить), replace(), find(), count(), len(s). Срезы: s[1:4], s[::-1] (разворот).
5. Функции (методы)
Функция — блок кода, который можно вызывать по имени. def name(args): return .... Параметры — входные данные, return — результат. Метод — функция, привязанная к объекту/классу.
6. Файловый ввод-вывод. Работа с файлами
Открыть: open("f.txt", "r") (r-чтение, w-запись, a-добавить). Лучше через with open(...) as f: — сам закроет. Чтение: f.read(), f.readlines(). Запись: f.write("текст").
7. Класс как абстрактный тип. Создание и использование
Класс — чертёж для объектов, объединяет данные (поля) и поведение (методы). class Dog: def __init__(self, name): self.name = name. Создание объекта: d = Dog("Рекс"). Это абстракция — описываешь «что такое собака» в общем.
8. Использование объектов и классов
Объект — конкретный экземпляр класса. Создаём: obj = MyClass(). Обращение к полям/методам через точку: obj.field, obj.method(). Один класс → много объектов с разными данными.
9. Списки, кортежи, словари
Список [1,2,3] — изменяемый, упорядоченный. Кортеж (1,2,3) — НЕизменяемый. Словарь {"ключ":"значение"} — пары ключ-значение, доступ по ключу.
10. Компоненты для работы с текстом
Строковые методы (split, join, replace, format), f-строки (f"{x}"), модуль re (регулярные выражения), модуль string. В Tkinter — виджеты Entry, Text, Label.
11. Модули. Определение и подключение
Модуль — файл .py с функциями/классами, который можно переиспользовать. Подключение: import math, from math import sqrt, import numpy as np. Свой модуль: создаёшь myfile.py и пишешь import myfile.
12. Как принять ввод с клавиатуры?
Функцией input(): x = input("Введите: ") — вернёт строку. Для числа оборачивай: int(input()) или float(input()).
13. Как проверить, что все символы строки алфавитно-цифровые?
Методом s.isalnum() — вернёт True, если все символы буквы или цифры. Похожие: isalpha() (только буквы), isdigit() (только цифры).
14. Разница между списком и кортежем
Список (list, []) — изменяемый: можно добавлять/удалять элементы. Кортеж (tuple, ()) — неизменяемый: после создания не поменять. Кортеж быстрее и его можно использовать как ключ словаря.
15. Функция range
Генерирует последовательность чисел. range(5) → 0,1,2,3,4. range(2,8) → 2..7. range(0,10,2) → 0,2,4,6,8 (шаг 2). Часто в циклах for.
16. Локальные и глобальные переменные
Локальная — создана внутри функции, видна только там. Глобальная — вне функций, видна везде. Чтобы менять глобальную внутри функции — ключевое слово global x.
17. Разница между func и func()
func — ссылка на саму функцию (объект), можно передать куда-то. func() — вызов функции, выполняет её и возвращает результат.
18. Как работает функция map
map(функция, список) применяет функцию к каждому элементу. Пример: map(str, [1,2,3]) → "1","2","3". Результат оборачивают в list(): list(map(...)).
19. По ссылке или по значению?
В Python — «по ссылке на объект» (точнее, передаётся ссылка). Изменяемые объекты (списки) можно менять внутри функции, и снаружи это видно. Неизменяемые (числа, строки) — нет.
20. Как развернуть список?
Срезом: arr[::-1]. Или методом arr.reverse() (меняет сам список). Или list(reversed(arr)).
21. Что означает self в классе?
self — ссылка на сам объект. Через него методы обращаются к полям этого объекта: self.name. Первый параметр каждого метода класса.
22. Как объединить списки?
Оператором +: a + b. Методом a.extend(b). Или распаковкой: [*a, *b].
23. Разница между списками и массивами
Список (list) может хранить разные типы (число, строку вместе), встроен в Python. Массив (array/numpy) хранит элементы одного типа, занимает меньше памяти, быстрее для вычислений. Numpy-массивы для математики.
24. Изменяемые и неизменяемые объекты
Изменяемые: list, dict, set. Неизменяемые: int, float, str, tuple, bool. Неизменяемые нельзя поменять «на месте» — создаётся новый объект.
25. Как округлить до трёх знаков?
Функцией round(x, 3). Например round(3.14159, 3) → 3.142. Или форматом f"{x:.3f}".
26. Где быстрее поиск: в словарях или списках?
В словарях. Словарь использует хеш — поиск по ключу почти мгновенный (O(1)). В списке поиск перебором (O(n)) — медленнее на больших данных.
27. Разница между модулем и пакетом
Модуль — один файл .py. Пакет — папка с модулями (содержит файл __init__.py). Пакет группирует связанные модули.
28. Как увеличить/уменьшить целое число?
x += 1 (увеличить на 1), x -= 1 (уменьшить). В Python нет x++ как в Си.
29. Как проверить, есть ли значение в списке?
Оператором in: if 5 in arr: → True/False.
30. Разница между append и extend
append(x) добавляет x как ОДИН элемент (даже если это список — вложит целиком). extend(list) добавляет каждый элемент списка по отдельности.
31. Разница между pass, continue и break
pass — ничего не делает (заглушка). continue — пропускает остаток итерации, идёт к следующей. break — полностью выходит из цикла.
32. Какие типы данных в Python? (5)
int (целые), float (дробные), str (строки), bool (логические), list (списки). Ещё: tuple, dict, set, complex.
33. Обработка исключений
Блок try / except: try: рискованный код except ОшибкаType: что делать. Ещё else (если ошибки не было) и finally (выполнится всегда).
34. Tkinter: окно, кнопки, позиционирование
Окно: win = tk.Tk(), размер win.geometry("300x200"), запуск win.mainloop(). Кнопка: tk.Button(win, text="OK", command=func). Позиционирование: pack(), grid(row,column), place(x,y).
35. Tkinter: Label, Entry и другие элементы
Label — текстовая метка. Entry — поле ввода (одна строка), читаем ent.get(). Text — многострочное поле. Checkbutton, Radiobutton, Listbox, Canvas (графика).
36. Работа с базой SQLite
import sqlite3. Подключение: con = sqlite3.connect("db.db"), курсор cur = con.cursor(). Запросы: cur.execute("SELECT ..."), сохранить con.commit(), закрыть con.close().
37. Python полностью поддерживает ООП?
Да. Есть классы, объекты, наследование, инкапсуляция, полиморфизм. Но инкапсуляция «по соглашению» (приватность через _ и __, не строгая, как в Java).
38. Что может быть ключом в словаре?
Только неизменяемые (хешируемые) объекты: строки, числа, кортежи. Список или другой словарь ключом быть не могут.
39. Для чего функция __init__?
Это конструктор — вызывается автоматически при создании объекта. Задаёт начальные значения полей: def __init__(self, name): self.name = name.
40. Синхронный и асинхронный код
Синхронный — команды выполняются по очереди, каждая ждёт предыдущую. Асинхронный — может начать задачу и не ждать (async/await), пока идёт ожидание (например запрос в сеть) — делает другое. Асинхронный эффективнее при ожиданиях.

☕ Java (41–46)

41. Основные отличия Kotlin и Java
Kotlin современнее и короче: null-безопасность (?), нет точек с запятой, val/var вместо типов, data-классы. Java старше, многословнее, но огромная база кода. Оба работают на JVM, Kotlin — официальный язык Android.
42. Назначение Java. Средства разработки
Java — кроссплатформенный язык («написал раз — работает везде» через JVM). Применение: серверы, Android, десктоп. Средства: JDK (комплект разработчика), IDE (IntelliJ IDEA, Eclipse), компилятор javac.
43. Типы данных и операции (Java)
Примитивы: int, double, float, char, boolean, long, short, byte. Ссылочные: String, массивы, объекты. Операции: арифметические (+ - * / %), сравнения (== != < >), логические (&& || !).
44. ООП в Java
4 принципа: Инкапсуляция (скрытие данных, private + геттеры/сеттеры), Наследование (extends), Полиморфизм (один интерфейс — разные реализации), Абстракция (abstract, interface).
45. Работа со строками (Java)
Класс String (неизменяемый). Методы: length(), charAt(), substring(), indexOf(), equals() (сравнение!), toUpperCase(), replace(). Для частых изменений — StringBuilder.
46. Геттеры и сеттеры в Java
Поля делают private, доступ через методы. Геттер getName() — возвращает значение. Сеттер setName(value) — устанавливает. Это инкапсуляция: контроль доступа к данным.

🤖 Android Studio (47–60)

47. Что такое Activity? Жизненный цикл
Activity — один экран приложения. Жизненный цикл (методы): onCreate (создан) → onStart (виден) → onResume (активен) → onPause → onStop → onDestroy. Система вызывает их сама при открытии/сворачивании.
48. Что такое AndroidManifest?
Главный файл-описание приложения (AndroidManifest.xml). Содержит: имя пакета, список Activity, разрешения (permissions), иконку, какая Activity стартовая. Без него приложение не соберётся.
49. Что такое Intent?
Intent («намерение») — объект для запуска действий: открыть другую Activity, позвонить, открыть камеру, передать данные. Явный (указываешь класс) и неявный (указываешь действие, систему сама выбирает приложение).
50. Диалоговые окна Android
AlertDialog (сообщение с кнопками OK/Cancel), DatePickerDialog (выбор даты), TimePickerDialog (время), ProgressDialog (загрузка), Toast (всплывающее сообщение, технически не диалог).
51. Виды Layout
LinearLayout — в ряд/столбец. RelativeLayout — относительно друг друга. ConstraintLayout — гибкий, по «привязкам» (сейчас основной). FrameLayout — слоями. TableLayout — таблицей.
52. Исполняемые файлы на Android
APK (Android Package) — установочный файл приложения. AAB (Android App Bundle) — формат для Google Play, из него генерируются оптимизированные APK. Внутри: код (DEX), ресурсы, манифест.
53. Что такое View? Примеры ViewGroup
View — базовый элемент интерфейса (кнопка, текст, картинка). ViewGroup — контейнер для View (сам тоже View). Примеры ViewGroup: LinearLayout, RelativeLayout, ConstraintLayout, FrameLayout, RecyclerView.
54. Структура проекта в Android Studio
manifests/ — AndroidManifest.xml. java/ (или kotlin) — код (Activity). res/ — ресурсы: layout (разметка), drawable (картинки), values (строки, цвета). Gradle — файлы сборки (зависимости).
55. Что такое SDK Manager и AVD Manager?
SDK Manager — управляет версиями Android SDK (инструменты, платформы). AVD Manager — создаёт и управляет виртуальными устройствами (эмуляторами), на которых тестируешь приложение.
56. Создание простого меню
Создаёшь файл res/menu/menu.xml с пунктами (item). В Activity переопределяешь onCreateOptionsMenu (надуть меню) и onOptionsItemSelected (обработать нажатие пунктов).
57. Папка res/values. Ресурсы приложения
Хранит ресурсы в XML: strings.xml (тексты), colors.xml (цвета), styles/themes.xml (стили), dimens.xml (размеры). Доступ из кода: R.string.app_name. Удобно для перевода и единообразия.
58. Среды разработки для Android
Главная — Android Studio (официальная, на IntelliJ IDEA). Другие: Eclipse + ADT (устарела), Visual Studio (Xamarin), Flutter/React Native (кроссплатформа), IntelliJ IDEA.
59. Адаптеры и списки в Android
Адаптер связывает данные с виджетом-списком. ArrayAdapter — простой список из массива. BaseAdapter — свой. Списки: ListView (старый), RecyclerView (современный, эффективный для длинных списков).
60. Работа с графикой и изображениями
ImageView — показ картинок (из res/drawable). Canvas + Paint — рисование (линии, фигуры). Библиотеки загрузки картинок из сети: Glide, Picasso. Форматы: PNG, JPG, WebP, векторные XML (vector drawable).
